# Test-Driven Development (TDD)

## The Cycle
1. **Red**: Write a minimal test that fails for the expected reason.
2. **Green**: Write the minimum amount of code to make the test pass.
3. **Refactor**: Clean up the code while keeping the test passing.

## Guidelines
- Never write implementation code without a failing test.
- Tests should be fast and focused.
- Ensure the test fails for the *right* reason (e.g., if you expect an error, verify the error message).
